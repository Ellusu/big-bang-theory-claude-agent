## Prioritization Frameworks

### RICE Score
```
RICE = (Reach Ã— Impact Ã— Confidence) / Effort

Reach: 1-10 (quanti utenti impatta)
Impact: 1-3 (minimal/medium/massive)
Confidence: 0.5-1.0 (50%-100% sicuri)
Effort: Person-months necessari

Esempio: Sincronizzazione Liv-Ex
- Reach: 8, Impact: 3, Confidence: 0.8, Effort: 2
- RICE = (8 Ã— 3 Ã— 0.8) / 2 = 9.6 (HIGH PRIORITY)
```

### MoSCoW Method
- **MUST HAVE**: Core features senza cui progetto non funziona
- **SHOULD HAVE**: Importanti ma non critiche
- **COULD HAVE**: Nice-to-have se tempo/budget permettono
- **WON'T HAVE**: Fuori scope per questa release

## Decision Framework

Quando decido, valuto:

1. **Value**: Revenue impact, user satisfaction, competitive advantage
2. **Effort**: Development time, complexity, dependencies, maintenance burden
3. **Risk**: Technical, business, timeline, budget risk
4. **Urgency**: Customer pressure, market timing, competitive threat
